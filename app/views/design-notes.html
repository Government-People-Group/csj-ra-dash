V2


Aim

V2 was an updated version of V1, based on the UR feedback received for V1. Our aim was to:

  • remove the banked score tab that was considered unnecessary
  • change some misleading content e.g. 'timeline' to 'case history'
  • change button content so users had less fear about what the button would do (case overview tab)
  • add extra contact options for the caseworker
  • add missing reference numbers
  • change the 'urgent' tag to something more meaningful
  • hide completed cases

What we did

We made the above changes to create a V2 version of the dashboard.

V1


Aim

Following on from a workshop with RAST leadership team, our aim was to:

  • provide RAST with a dashboard for handling RA cases
  • allow case lists to be viewed and filtered
  • show MI for leadership staff
  • provide RA guidance for caseworkers
  • prioritise important / urgent cases
  • show the info needed to allow a caseworker to progress an RA case

What we did

We designed a dashboard, with:

  • an MI homepage which displayed stats and links through to the subgroup of cases
  • a list of cases
  • details on each case when clicked into

User research

Mural board with testing notes


What we found

  • the dashboard was well received overall
  • the filters were considered useful, although some refinement is needed as to what filters would be needed in reality
  • users were confused as to what the 'URGENT' tag meant for that particular case
  • it was felt that some reference numbers were missing
  • the tabbed case details was successful and easily navigated
  • it was felt that the banked scores tab was not needed, as no users felt they needed to see the actual banked score
  • users showed reticence to use the options like 'reject request', as they felt the action would happen immediately (in actual fact, the plan was to take them to further screens before this would be final)
  • the 'Case notes' tab was well received, although some users were unsure of it's purpose at first, while others raised concerns over who could / couldn't see the notes
  • 'Timeline' proved a confusing tab name, as some users thought it would show what would happen when, rather than be an audit trail as planned
  • seeing previous requests by that candidate was considered very useful
